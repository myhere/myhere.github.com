同步 or 异步

  • 开发模式
  • 维护成本
  • 体验

同步开发流程

  • 前端写 demo
  • 开发套页面, vm. 数据
  • 联调出 bug
    • 改 demo
    • 改 vm

同步开发中的问题!

  • demo 要经过开发修改
  • 前后端分离不彻底
  • 前端没有独立开发环境
  • java 开发环境前端搭建成本高

异步开发模式

  • 前端写 html
  • 后端提供数据
  • 联调数据接口
  • 联调出 bug
    • 后端盖接口
    • 前端改 html

异步开发优点

  • 前后端分离
  • 前端有独立开发环境
  • 响应式
  • 图片retina
  • 多终端

异步实现方案

  • 前端方案
    • ui 模块化
    • ui 模块独立, 可插拔
    • ui 模块 assets 线上 combo, 按需加载
  • 数据接口
    • 字段与 ui 模块的约定
    • 一个数据字段对应一个 ui 模块

前端设计

app + mods

mod 都是相同的, 不知道彼此存在

app

  • 渲染页面各个容器
  • 管理页面中 ui 模块, 调用 mod.render
  • 模块间通信机制: 广播/收听
  • 异步点击事件绑定
    • 代理绑定, mod 调用绑定函数
    • 约定绑定, .J_Ajax
    • 参数与接口约定
  • history 管理
  • 请求数据
  • 其他公共功能

mods

  • doRender 渲染自己
  • 包括
    • js
    • css
    • html
  • 组织 => 内聚
    • 每个模块一个入口文件 index.js
    • index.js require css
    • index.js require html
  • 一个模块对外只公布一个入口文件

前端渲染流程

  • 初始化 app 对象
  • app 对象写入模块占位符
  • 初始化各个 mod 对象, 并挂到 app 对象上去
  • app 请求接口
  • 数据回来后 app 调用 mod.render
  • mod.render 渲染自己

优化

  • 性能统计: 模块渲染时间
  • 模板模块化: 预编译
  • 渲染优化: innerHTML 最耗时
    • 页面渲染分段
    • 组件内部分次渲染
  • js dom 操作
    • #idSelector
    • 缓存 dom 查找
    • 缓存 show/hide
  • 容错
    • 一个 mod 报错, 不影响其他 mod
    • jstracker
