怎么做

开发环境

  • 本地预览
  • 模拟 tms 环境, 借用 sdk api
  • svn管理, scm 监控 svn
  • checkout 直接开发

开发

  • 模块化

    • 头部
    • 商品列表
    • 浮动导航
    • 秒杀
    • banner
    • 会场互串
  • 皮肤: css 覆盖 or php 变量
  • 模块组合页面
  • 修改放到 tms

文件组织

优点:

  • 方便统一控制
  • 公用头尾修改容易
  • 页面本身代买修改容易
  • 添加新页面容易

代码

  • 功能按钮等用钩子, 全局统一初始化(保证不报错)
  • 内容在一个地方填写, 不划分区块 include

  • 内容展现分离 秒杀内容

    • 方便统一修改 ui

性能优化

  1. 全局统一 lazyload
  2. ie6 提示是否终止脚本
  3. Chrome

一些经验

  • 不要用图片文字(楼层), 微软雅黑代替
  • 文案不要切图片

不足

  • 无法完全模拟 tms 环境, 路径不一致, /home/admin

  • 没做延迟渲染, 对比去年双12, 渲染大概慢 100ms

    • performance

    • 2011-12-12 2012-11-11 

    • performance.timing.domInteractive - performance.timing.domLoading
