文件组织-好处

  • 方便统一控制
  • 公用头尾修改容易
  • 页面本身代码修改容易
  • 添加新页面容易

遇到的问题

  1. tms 生成预热页面总是失败

    设置 php 执行环境

    ini_set('memory_limit', -1);
ini_set('max_input_time', 100000);
ini_set('max_execution_time', 100000);
set_time_limit(0);

  2. 部分页面收藏失败 debug
  3. ie6 提示是否终止脚本 profile

一些经验

  • 功能按钮等用钩子, 全局统一初始化(保证不报错)
  • 内容在一个地方填写, 不划分区块 include

  • 内容展现分离 秒杀内容

    • 很多区块填数据, 一个区块负责输出
    • 方便统一修改 ui
  • tms 1个小时缓存, 秒杀异步加载
  • 不要用图片文字(楼层), 微软雅黑代替
  • 文案不要切图片

不足

  • 无法完全模拟 tms 环境, 路径不一致, /home/admin

  • 没做延迟渲染, 对比去年双12, 渲染大概慢 100ms

    • performance

    • 2011双12 vs 2012双11 

    • performance.timing.domInteractive - performance.timing.domLoading
